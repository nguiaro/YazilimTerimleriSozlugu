## A

**A/B Testing** = Comparative testing of two different versions (A and B) of a web page or application to measure user experience.
<hr>

**Abort** = The forced stopping of a process that has already started (e.g., file download or database query) before it is completed.
<hr>

**Absolute Path** = Specifying the full address of a file or resource starting from the root directory (e.g., `C:/Users/Documents/file.txt` or `https://site.com/image.jpg`).
<hr>

**Abstract Class** = In Object-Oriented Programming (OOP), a template class from which objects cannot be directly created, but which is used to provide inheritance to other classes.
<hr>

**Abstraction** = The principle of making a system more understandable by hiding complex system details and exposing only the necessary functions.
<hr>

**Acceptance Testing** = The final stage test performed to verify whether the software meets customer or user requirements.
<hr>

**Access Control** = A security mechanism that determines who can access which resource (Usually done as Role-Based).
<hr>

**Access Modifier** = Keywords that determine how visible/accessible classes or variables are from the outside (Private, Public, Protected).
<hr>

**Access Token** = A digital key (usually in JWT format) used by a user in API requests to prove authorization after logging into the system.
<hr>

**Accessibility / A11y** = Standards ensuring that software is usable by everyone, including individuals with disabilities.
<hr>

**ACID (Atomicity, Consistency, Isolation, Durability)** = Four fundamental properties that guarantee the reliability of database transactions.
<hr>

**Action** = In HTML forms, it specifies the address where data will be sent (`action="/login"`); in contexts like Redux, it is a packet of information sent to change the application's state.
<hr>

**Active Record** = A design pattern that converts rows in database tables into objects and facilitates database operations.
<hr>

**Adapter Pattern** = A design pattern that makes two incompatible interfaces or classes work together by placing a translator (adapter) between them (logic of a plug adapter).
<hr>

**Add-on** = An external module that adds new features to a software (usually browsers) without changing its core structure. (Used synonymously with Plugin).
<hr>

**Address** = A numerical value indicating the location of data in memory (RAM).

<hr>

**Admin / Administrator** = The top-level user role that has all permissions in the system, can change settings, and manage other users.
<hr>

**Adware** = A type of annoying software that constantly displays advertisements on the user's screen without their consent, usually transmitted via free software.
<hr>

**Agile** = A project management methodology that aims to respond quickly to change and progress in iterations.

<hr>

**AI (Artificial Intelligence)** = Computer systems capable of learning, problem-solving, and decision-making by mimicking human intelligence.
<hr>

**AJAX (Asynchronous JavaScript and XML)** = A technique that allows updating a page by exchanging data with the server in the background without reloading the entire page.
<hr>

**Alert** = A simple message window, especially in JavaScript, that suddenly appears on the user's screen and does not close until "OK" is pressed (`alert("Hello")`).
<hr>

**Algorithm** = A set of logical and sequential steps followed to solve a problem.

<hr>

**Alias** = A temporary or alternative name given to a variable, command, or database table.
<hr>

**Allocation** = The process by which the operating system assigns the memory (RAM) or disk space required for a program to run.
<hr>

**Alpha** = The first, very raw version of software that begins testing, full of bugs. (Order: Alpha -> Beta -> Release Candidate -> Stable).

<hr>

**Analog** = An older technology or signal type where data is transmitted in continuous waves, not digital (0 and 1).

<hr>

**Analytics** = The process of measuring application or website traffic and user behavior and reporting it with graphs (e.g., Google Analytics).
<hr>

**Anchor** = A connection (Link) created with the `<a>` tag in HTML that allows navigating to another page or a location within the page.
<hr>

**Android** = The Linux-based, open-source mobile operating system developed by Google, which is the most used in the world.
<hr>

**Angular** = A popular TypeScript-based Frontend framework developed by Google, preferred especially in large corporate projects (One of React's biggest rivals).
<hr>

**Annotation** = Metadata written into the code that does not directly affect the program's execution but gives information to the compiler or other tools (e.g., `@Override` in Java).
<hr>

**Anonymous Function** = A function that does not have a name, usually sent as a parameter into another function or used once.
<hr>

**Anti-Pattern** = "Bad solution" methods often used in software development that are actually inefficient, erroneous, or cause problems in the future. (e.g., Writing spaghetti code is an anti-pattern).
<hr>

**Apache** = One of the world's most established and widely used open-source web server software.
<hr>

**API (Application Programming Interface)** = An interface that allows different software to talk to each other.

<hr>

**App Store** = A digital market where mobile or desktop applications are distributed, and users can download or purchase them.
<hr>

**Append** = The process of adding new data to the very end of a file, list, or text. (e.g., You append a new message to the chat history).
<hr>

**Applet** = A small Java application that usually runs within a browser (Its usage has decreased today).
<hr>

**Application Server** = An advanced type of server that handles not only static pages (HTML) but also complex business logic and dynamic processes.
<hr>

**Architecture** = The general structure of a software system's components, the relationships between these components, and design principles.

<hr>

**Archive** = A file format where files are stored by being compressed or grouped together (zip, tar, rar, etc.).
<hr>

**Argument** = Data sent from the outside to functions for them to process.
<hr>

**Arithmetic Operators** = Symbols that allow mathematical operations (+, -, *, /).
<hr>

**Array** = A data structure that holds data in a sequential list.
<hr>

**Artifact** = A general term used for any output produced during the software development process (Documents, compiled .exe files, libraries).
<hr>

**ASCII** = A standard coding table used for computers to recognize characters.
<hr>

**Assembly** = The lowest-level programming language readable by humans, closest to machine code (0 and 1). It speaks directly to the hardware.
<hr>

**Assertion** = A logical expression that assumes a condition must be "true" at a specific point in the code; frequently used in testing processes.
<hr>

**Asset** = Non-code resource files used in a software project, such as images, audio, video, or fonts.
<hr>

**Assignment** = The process of giving a value to a variable (Usually done with the `=` sign).
<hr>

**Asynchronous** = Execution of operations without waiting for each other, in a non-simultaneous manner.
<hr>

**Atomic Design** = A methodology suggesting starting interface design from the smallest parts (Atoms: button, input) and combining them to go to larger structures (Molecules -> Organisms -> Pages).
<hr>

**Attribute** = Parameters that define additional properties for HTML tags or objects.
<hr>

**Audit Log** = A "black box" file that records step-by-step who did what operation and when in the system (for security purposes).
<hr>

**Augmented Reality / AR** = Technology that adds computer-generated graphics over the real-world view via a camera or glasses (e.g., Pokemon GO).

<hr>

**Authentication** = The process of verifying a user's identity (Login process).
<hr>

**Authorization** = Determining which resources a user can access (Permission check).
<hr>

**Auto-Complete** = A feature where the system predicts and suggests possible words while typing in a search box.
<hr>

**Auto-Increment** = In a database, when a new record is added, the ID number increases automatically (1, 2, 3...) without our intervention.
<hr>

**Auto-Scaling** = In cloud computing, the process where servers automatically increase (scale out) or decrease (scale in) based on demand/traffic.
<hr>

**Automation** = The process of making manual operations automatic through software.
<hr>

**Availability** = A metric indicating how long a system remains operational without interruption.
<hr>

**Avatar** = A small profile picture or graphic character representing the user.
<hr>

**Await** = A command used inside `async` functions in JavaScript that says, "Don't move to the next line until this process is finished, wait."
<hr>

**AWS (Amazon Web Services)** = The world's most popular cloud computing service provider offered by Amazon.

<hr>

**Azure** = A massive cloud computing platform offered by Microsoft, a rival to AWS and Google Cloud.
<hr>

## B

**Backend** = The server-side of a website or application. Database operations, server settings, and main logical processes that the user does not see happen here.

<hr>

**Backdoor** = An entry point secretly left by a developer or attacker to bypass normal security procedures (password, etc.) to enter a system.
<hr>

**Background Job** = Long processes run quietly by the server in the background without making the user wait (e.g., "Prepare the report and send an email").
<hr>

**Backlog** = A list, usually used in Agile methodology, containing features, fixes, and tasks that need to be done during the project, ordered by priority.

<hr>

**Backup** = The process of storing a copy of data in a safe place against the risk of data loss or corruption.
<hr>

**Bad Request** = HTTP 400 error code indicating that the server could not understand the request coming from the client (browser) or that the request format is malformed.
<hr>

**Badge** = A small red circle usually appearing on the corners of icons, showing the number of unread messages or notification status.
<hr>

**Bandwidth** = The maximum amount of data that can be transmitted over a network connection in a specific time.
<hr>

**Bare Metal** = A server structure where software runs directly on hardware without a virtualization layer or operating system.
<hr>

**Base Case** = The termination condition in recursive functions that stops them from running infinitely, saying "Stop now."
<hr>

**Base URL** = The unchanging root address of a website or API (e.g., the `api.google.com` part in `https://api.google.com/v1/...`).
<hr>

**Base64** = A coding scheme that allows transporting binary data (images, files, etc.) by converting it into text format.
<hr>

**Bash (Bourne Again Shell)** = A popular command-line interface and shell program used in Linux and Unix-based systems.
<hr>

**Bastion Host** = A security-hardened server used as a "jump point" to securely connect to a private and closed network.
<hr>

**Batch Processing** = Processing accumulated data or tasks in groups one after another without human intervention.
<hr>

**Bearer Token** = A security key sent in the Authorization header in API accesses, working on the logic of "Open the door to the person holding this ticket."
<hr>

**Benchmark** = The process of measuring the performance (speed, power) of hardware or software with standard tests and comparing it with others.
<hr>

**Best Practice** = The method accepted as the most correct, secure, and efficient way of doing a job in the software world, proven by experience.
<hr>

**Beta** = The version of software in the development stage, not yet the final release, but offered to a limited audience for testing.
<hr>

**Big Data** = Very large-volume, fast, and diverse data sets that are difficult to process with traditional methods.
<hr>

**Big O Notation** = Mathematical notation used to analyze the efficiency and performance (speed or memory usage) of an algorithm based on input size.
<hr>

**Binary** = The numeral system consisting only of 0 and 1 digits, which is the basic language of computers.
<hr>

**Binary Search** = A very fast search algorithm that finds a value in a sorted list by repeatedly dividing the list in half.
<hr>

**Binary Tree** = A hierarchical data structure where each node has at most two branches (left and right).

<hr>

**Binding** = The process of connecting a function or variable to a specific object or scope (Related to the `this` keyword in JavaScript).
<hr>

**Bios (Basic Input/Output System)** = Small software on the motherboard that tests hardware and starts the operating system when the computer is turned on.
<hr>

**Bit** = The smallest unit of data in a computer (0 or 1).
<hr>

**Bitbucket** = A popular Git-based code hosting service owned by Atlassian, a rival to GitHub and GitLab.
<hr>

**Bitmap (BMP)** = An image format created by holding the color information of each pixel of the image.
<hr>

**Bitwise Operation** = Low-level mathematical operations that process data directly at the level of 0s and 1s (bit by bit), not as bytes or numbers.
<hr>

**Black Box Testing** = A testing method where the tester checks the "Result" by giving "Input" without knowing the internal structure of the code.

<hr>

**Black Hat Hacker** = A malicious cyber pirate who uses their skills to damage systems, steal data, or commit blackmail.
<hr>

**Blob (Binary Large Object)** = A chunk of raw data, usually large in size like images, audio, or video, stored in a database or memory.
<hr>

**Block** = A group of code usually written between curly braces `{ }` and accepted as a single unit.
<hr>

**Block Scope** = The situation where a variable is valid only within the curly braces `{ }` where it is defined (`let` and `const` in JavaScript are block-scoped).
<hr>

**Blockchain** = A decentralized and immutable ledger technology where data is stored in encrypted chained blocks.

<hr>

**Blocking** = Code structure that does not allow another process to start before one finishes, locking the system (Synchronous operation).
<hr>

**Blue Screen of Death / BSOD** = The famous blue error screen that appears when Windows operating system encounters a critical error and shuts itself down for protection.
<hr>

**Blur** = Loss of sharpness in visual design; In UI, it is the event of the cursor leaving an input box (focus out).
<hr>

**Body** = The main tag covering all content seen by the user in an HTML page `<body>`; In an HTTP request, it is the part where sent data (e.g., form information) is carried.
<hr>

**Boilerplate Code** = Standard pieces of code used repeatedly in software projects with little or no changes.
<hr>

**Boolean** = A logical data type. It can only take two values: True (1) or False (0).
<hr>

**Bookmark** = The process of saving a website's address in the browser to find it easily later.
<hr>

**Boot** = The process of starting a computer or an operating system.
<hr>

**Bootstrap** = The world's most popular CSS library used for designing fast and responsive (mobile-compatible) websites.
<hr>

**Bot** = Software or script that automatically performs specific tasks (e.g., Search engine bots, Chatbots).
<hr>

**Bottleneck** = The weakest link that lowers the overall performance of a system and slows down the flow (e.g., If you have a fast processor but slow internet, the internet is a bottleneck).
<hr>

**Box Model** = The basis of CSS. It states that every HTML element is a box consisting of Margin, Border, Padding, and Content layers.
<hr>

**Branch** = In version control systems like Git, an independent workspace opened to develop different features without affecting the main project.
<hr>

**Breadcrumb** = A horizontal navigation bar on websites showing where the user is and how to go back (e.g., Home > Electronics > Phone > iPhone).
<hr>

**Break** = A command used in loops or Switch structures to interrupt the process and exit.
<hr>

**Breakpoint** = A marker that pauses the program when it reaches that line during debugging.
<hr>

**Broadcast** = Sending data on a network to all devices connected to the network simultaneously.
<hr>

**Broken Link** = A non-working link that gives a 404 error when clicked because the target page cannot be found.
<hr>

**Browser** = Software that allows viewing websites (Chrome, Firefox, Safari, etc.).
<hr>

**Brute Force** = An attack or solution method that tries every possible combination one by one to find a password or key.
<hr>

**Bubble Sort** = A simple sorting algorithm frequently shown in education but rarely used in real life due to low performance.
<hr>

**Bubbling** = In JavaScript, when an inner element is clicked (e.g., a button), this click event spreads upwards (to the div containing the button, then to the body).
<hr>

**Buffer** = A memory area where data is temporarily held before being processed during data transfer.
<hr>

**Buffer Overflow** = An attempt to crash or infiltrate a program by sending more data than the allocated memory area can handle.
<hr>

**Bug** = A code error that causes software to behave unexpectedly, produce wrong results, or crash.
<hr>

**Build** = The process of converting source code into an application that the computer can run.
<hr>

**Bundle** = A single file created by tools like Webpack by combining and compressing hundreds of JavaScript files to be sent to the browser (e.g., `main.bundle.js`).
<hr>

**Business Logic** = The code part where main business rules like "Credit calculation rule" or "Discount application condition" are written, not database or interface.
<hr>

**Byte** = A unit of data consisting of 8 bits, usually representing a character.
<hr>

## C

**Cache** = Holding frequently used data temporarily in a fast storage area so that it can be accessed faster.
<hr>

**Callback** = A function sent as a parameter to another function to be executed when a process is completed (Very frequently used in JavaScript).
<hr>

**Callback Hell** = The situation in JavaScript where nested functions make the code unreadable and create a "pyramid extending to the right" while managing asynchronous operations.
<hr>

**Camel Case** = One of the variable naming standards. The first word is lowercase, and the first letters of subsequent words are uppercase (e.g., `userName`, `totalPrice`).
<hr>

**Canvas** = An area introduced with HTML5 where drawings (graphics, games, animations) can be made using JavaScript.
<hr>

**Captcha** = A test used on websites to verify that the person filling out the form is a real human, not a robot (bot) (e.g., "Select traffic lights").
<hr>

**Card** = A popular UI component in modern web design where content (image, title, summary) is grouped inside a rectangular box.
<hr>

**Cardinality** = The uniqueness rate of data in a database table column. (e.g., Gender column has low cardinality [only Female/Male], ID No column has high cardinality [everyone is different]).
<hr>

**Carousel** = Interface element where images or content slide side by side (slider) on websites.
<hr>

**Cast / Casting** = The process of forcibly converting one data type to another (e.g., Converting text "10" to number 10).
<hr>

**Catch** = The block in a Try-Catch structure that catches the error and specifies what to do so the program doesn't crash when an error occurs.
<hr>

**CDN (Content Delivery Network)** = A network of servers that ensures website content (images, videos, etc.) is delivered quickly to users from the nearest server.
<hr>

**Certificate** = A digital identity file issued by authorized institutions proving that websites are secure (HTTPS) (SSL Certificate).
<hr>

**Change Log** = A list explaining item by item what changed and which errors were fixed in each new version of software.
<hr>

**Character / Char** = A data type representing a single letter, number, or symbol (e.g., â€˜Aâ€™, â€˜9â€™, â€˜@â€™).
<hr>

**Charset** = The character table used by computers to store text (e.g., UTF-8). If chosen incorrectly, special characters appear broken.
<hr>

**Checkbox** = A square box where the user can select multiple options at the same time (e.g., Hobbies: [x] Music [ ] Sports [x] Cinema).
<hr>

**Checksum** = A mathematical summary value generated from file content to detect if a file has been corrupted during transfer.
<hr>

**Child / Child Element** = In HTML or data structures, a sub-element found inside another element (Parent).
<hr>

**Chunk** = Each of the small pieces into which large data or files are divided for easier processing or transport.
<hr>

**CI/CD (Continuous Integration / Continuous Deployment)** = The complete automation of the process from writing code to deploying it to live (testing, compiling, uploading). It is the backbone of modern software.

<hr>

**Cipher** = Mathematical formula or method used to encrypt and decrypt texts.
<hr>

**Circular Dependency** = The situation where file A needs B, and file B needs A again. It leads to program lock-ups or errors.
<hr>

**Class** = In Object-Oriented Programming (OOP), a template defining the properties and behaviors of objects (e.g., 'Car' is a class, 'Red BMW' is an object created from that class).
<hr>

**Clean Code** = A code philosophy written according to certain standards not just to work, but to be easy to read, understand, and maintain.
<hr>

**CLI (Command Line Interface)** = The screen where we manage the computer by typing text commands instead of a graphical interface (mouse/icons).
<hr>

**Clickjacking** = An attack where an invisible layer is placed over an innocent-looking button to trick the user into clicking something they didn't intend (e.g., a "Follow" button).
<hr>

**Client** = A user device or software requesting service or data from a server (e.g., Your web browser is a Client).
<hr>

**Client-Side** = Processes occurring on the user's own browser or device, not on the server (e.g., Animations made with JavaScript).
<hr>

**Clone** = Creating an exact copy of an object; In Git, downloading a remote project (Repository) to your own computer.
<hr>

**Closure** = One of JavaScript's most difficult-to-understand but powerful features. The situation where a function continues to remember variables in the environment where it was defined even after it finishes running.
<hr>

**Cloud Computing** = Providing computing services like data storage and processing power over the internet.

<hr>

**Cluster** = A group of servers or computers connected to work like a single system.
<hr>

**CMS (Content Management System)** = Software that allows creating websites and updating content without knowing code (e.g., WordPress, Joomla).
<hr>

**Code Review** = The process of checking written code by other developers to improve quality and find errors.
<hr>

**Code Splitting** = A performance technique, especially in structures like React, that allows loading pages piece by piece as the user needs them, instead of loading the entire application at once.
<hr>

**Cohesion** = How related and integrated the parts inside a code module are with each other. High Cohesion is desired (A class should focus on only one job).

<hr>

**Cold Start** = In Serverless architecture, the waiting time for a function to wake up and prepare when called for the first time after not running for a long time.
<hr>

**Collision** = In Hash algorithms, the situation where two different data accidentally produce the same digest value (Hash). Rare but risky.
<hr>

**Column** = Vertical fields in database tables. Each column holds a specific data type (e.g., "Surname" column).
<hr>

**Comment** = Notes written inside the code that are not executed by the computer but added only for developers to read.
<hr>

**Commit** = In systems like Git, permanently saving code changes with a message.
<hr>

**Compiler** = A program that translates written source code into machine language (0s and 1s) that the computer can understand.
<hr>

**Component** = Especially in modern web structures like React, reusable, independent small pieces of the interface (e.g., A 'Button' component).

<hr>

**Concatenation** = The process of combining two or more text pieces (strings) end-to-end to form a single text.
<hr>

**Concurrency** = The system's ability to handle more than one task in the same time interval. (Confused with Parallelism; Concurrency is switching between tasks quickly, Parallelism is doing them at the same time).
<hr>

**Condition** = The logic determining the flow of the program, "If this is true do this, else do that" (If/Else).
<hr>

**Config / Configuration** = Files holding settings determining how the software will run (e.g., `.env` files).
<hr>

**Console** = Text-based screen or developer tool used to display system messages, errors, or outputs.
<hr>

**Constant / Const** = A structure holding data whose value cannot be changed throughout the program execution.
<hr>

**Constructor** = The first function that runs automatically when a new object is created from a Class.
<hr>

**Container** = A unit where code and all necessary dependencies are packaged so that software runs smoothly in every environment (e.g., Docker).

<hr>

**Context** = In React, a method used to teleport data (theme, language, user info) from the top of the tree to the bottom without passing through intermediate components.
<hr>

**Cookie** = Small text files saved by websites to the browser to remember the user or store preferences.
<hr>

**Core** = Each independent brain inside processors (CPU) that processes commands (e.g., 8-Core Processor).
<hr>

**CORS (Cross-Origin Resource Sharing)** = A control mechanism browsers use for security when a site tries to fetch data from another site's API, asking "Do you allow this?". A common issue for developers.
<hr>

**Coupling** = How tightly two software modules depend on each other. Low Coupling is desired (So changing one doesn't break the other).
<hr>

**Crash** = A program stopping and closing due to an unexpected error.
<hr>

**Crawler** = Bots used by search engines (Google etc.) to visit pages on the internet and add them to the index.
<hr>

**Cron Job** = A scheduled task set up to run a specific command or script automatically at specific times (e.g., every morning at 09:00) on Linux/Unix systems.
<hr>

**CRUD (Create, Read, Update, Delete)** = The four basic operations of database applications.
<hr>

**CSS (Cascading Style Sheets)** = The language that handles the visual design (color, font, layout) of the skeleton created with HTML.
<hr>

**CSV (Comma-Separated Values)** = A very simple database/table format where data is stored as plain text separated by commas, readable by Excel.
<hr>

**Currying** = In functional programming, the technique of converting a function taking multiple parameters (`f(a,b,c)`) into a chain of nested functions each taking a single parameter (`f(a)(b)(c)`).
<hr>

## D

**Daemon** = A program running quietly in the background of the operating system performing specific tasks (e.g., checking emails, waiting for printer).
<hr>

**Dark Mode** = A theme option that displays the interface with a dark background and light text, reducing eye strain.
<hr>

**Dashboard** = A summary page where administrators can view and manage system status, data, and graphs on a single screen.
<hr>

**Data** = Raw pieces of information that the computer can process (Numbers, texts, visuals, etc.).
<hr>

**Data Lake** = A massive storage area where all kinds of structured (table) or unstructured (image, log) data are stored in their raw form without processing.
<hr>

**Data Mining** = The process of extracting hidden patterns and valuable information from large data piles using AI and statistical methods.

<hr>

**Data Science** = The discipline using statistics, mathematics, and software to extract meaningful information and patterns from large data sets.

<hr>

**Data Structure** = The method of organizing and storing data efficiently in computer memory (e.g., Array, List, Tree).
<hr>

**Data Type** = Classification indicating what kind of data a variable holds (e.g., Integer, Text, Date).
<hr>

**Data Warehouse** = A central system where data from different sources is cleaned, organized, and stored for reporting and analysis.

<hr>

**Database** = A digital repository where data is stored in an organized, secure, and quickly accessible way.
<hr>

**DDL (Data Definition Language)** = The sub-language of SQL covering table creation, deletion, and structure modification commands (CREATE, ALTER, DROP).
<hr>

**DDoS (Distributed Denial of Service)** = Sending attack traffic simultaneously from thousands of different computers (zombie devices) to crash a website or server.
<hr>

**Dead Code** = Forgotten and unnecessary code pieces found in the project that are never executed and take up space.
<hr>

**Deadlock** = The situation where two or more processes wait for each other to finish, so none can proceed and the system freezes. (e.g., Process A waits for B; B waits for A).
<hr>

**Debounce** = A performance technique especially in Frontend (React), where a function is executed only after the user stops an action (e.g., typing in a search box). "Don't search on every letter, do it when typing stops."
<hr>

**Debug** = The process of detecting, tracing, and fixing errors (bugs) in software.
<hr>

**Declaration** = Telling the compiler or interpreter that a new variable or function exists and what its name is.
<hr>

**Decryption** = The process of converting encrypted (unreadable) data back into its original and readable form using the correct key.
<hr>

**Deep Copy** = Copying an object not just by its outer shell but also by copying all sub-objects inside one by one. Even if the original object changes, the copy is not affected. (Opposite: Shallow Copy).
<hr>

**Deep Linking** = When a link is clicked, the mobile application opens not just the home page but a specific page inside (e.g., specific product detail).
<hr>

**Default** = The value or setting automatically assigned by the system when the user makes no special selection.
<hr>

**Definition of Done / DoD** = In Agile/Scrum teams, the list of criteria that must be met for a task to be considered "completed" (e.g., "Code written + Tested + Approved").
<hr>

**Workload** = Amount of work a server or system must process in specific time.
<hr>

**Wrapper** = Code layer wrapping complex code piece or library to make it simpler and understandable.
<hr>

**Write** = Process of saving data to disk, memory, or database.
<hr>

**Write-Only** = Access permission where data can only be written but not read (Log files are sometimes like this).
<hr>

**WWW (World Wide Web)** = Information system consisting of hypertext documents linked to each other with links published on internet. (Not internet itself, a service running on internet).
<hr>

## X

**XaaS (Everything as a Service)** = Offering all kinds of computing services via internet with subscription model regardless of software, infrastructure, or platform.
<hr>

**Xamarin** = Framework of Microsoft allowing developing mobile applications for both Android and iOS using C# (Rival of React Native).
<hr>

**Xcode** = Official development environment (IDE) used to develop applications for Apple devices (iPhone, Mac, iPad), running only on macOS.
<hr>

**XHR (XMLHttpRequest)** = Old generation JavaScript object used to fetch data from server in browsers (AJAX) before Fetch API came out.
<hr>

**XHTML (Extensible HyperText Markup Language)** = Old version of HTML based on XML having much stricter rules (every tag must close, lower case mandatory etc.).
<hr>

**XML (Extensible Markup Language)** = Markup language used to store and transport data, similar to HTML but where we name tags ourselves. (e.g., <student><name>Ali</name></student>).
<hr>

**XML Sitemap** = XML file containing list of all pages in a website telling search engines (Google) "My site has these pages, index them."
<hr>

**XOR (Exclusive OR)** = Logical operator. Works with logic "Either this or that; but not both." Result is true if only one of two inputs is true (1 XOR 0 = 1).
<hr>

**XP (Extreme Programming)** = Agile software methodology advocating frequent releases, pair programming, and continuous testing to increase software quality.
<hr>

**XPath** = Query language used to find or select a data inside XML files (XML version of QuerySelector in HTML).
<hr>

**XSS (Cross-Site Scripting)** = Attack of malicious people embedding harmful JavaScript codes inside a website to steal info (cookie, session) of other users visiting site.
<hr>

## Y

**YAGNI (You Aren't Gonna Need It)** = Important principle in software development. Means "Don't write code thinking it will be needed in future, write only what is needed now." Prevents unnecessary complexity.
<hr>

**YAML** = Data format usually used in configuration files (Docker, Kubernetes etc.), very easy to read. Has structure based on indentations (spaces).
<hr>

**Yarn** = Popular JavaScript package manager developed by Facebook as alternative to npm, ensuring faster and safer package downloading.
<hr>

**Yield** = Keyword used in languages like Python and C# for a function to produce current value and pause instead of finishing operation completely and returning result (return), then continue from where it left off (Generator functions).
<hr>

**Yottabyte** = Massive data unit. Equal to 1000 Zettabytes or 1 trillion Terabytes. (Big enough to cover all data on internet).
<hr>

## Z

**Z-Index** = Determines "depth" order of elements in CSS. Controls which box stands in front, which behind. As number grows element looks closer (above) to user.
<hr>

**Zero-Based Indexing** = Rule of starting counting from 0 not 1 in software world. First element of list is Array[0].
<hr>

**Zero-Day Vulnerability** = "Fresh" security vulnerability software developer is not yet aware of or hasn't patched, thus unprotected. Valuable as gold for hackers.
<hr>

**Zero-Knowledge Proof** = Method of proving you know something (e.g., your password) without telling that information itself to opposite party. Used in Cryptography and Blockchain.
<hr>

**ZIP** = Universal archive format used to compress files to reduce size and make single package.
<hr>

**Zombie Process** = "Dead but space occupying" process in operating system (Linux/Unix) whose execution finished but couldn't be completely deleted from memory.
<hr>

**Zone File** = Record file in DNS servers indicating which IP address a domain name goes to.
<hr>

**Zsh** = Modern terminal shell used in macOS and Linux, more advanced than standard Bash shell, having coloring and auto-completion features.
<hr>

**Zustand** = State Management library much simpler, lighter, and easier to install than Redux, whose popularity is increasing rapidly in React world recently.
